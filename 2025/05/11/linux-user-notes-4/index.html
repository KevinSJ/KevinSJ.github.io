Optimizing workflow with rofi, dmenu and desktop entry

Posted on 05-11-2025 Views:

Recently, I found myself needing to run virtual machines quite often, however, with two different types of VMs managed by different hypervisors on my machine, I find it quite inefficient to click through multiple hoops in GUI just to launch a single VM. Therefore, decided to spend time to find ways to improve this workflow.

The start: Controlling hypervisors from CLI and alias

As I mentioned previously, I ran two different types of Hypervisors on my machine:

  • VirtualBox (For Windows guests)
  • QEMU (For Linux guests)
    Let's dive into how to control the hypervisors from CLI

VirtualBox

In the past, I have attempted to start VirtualBox automatically on Windows host and wrote about it, this mostly involved running a batch script that can control the VirtualBox from the command line.

Quite conveniently, on Linux, we can use almost the same exact command to control things, so

  • To list all the available guests: vboxmanage list --sorted vms
  • To start a guest, we can use: vboxmanage startvm NAME_OF_GUEST
  • To power off a guest, we can use vboxmanage controlvm NAME_OF_GUEST poweroff

QEMU

To control guests managed by KVM, we will need to use virsh. Virsh is part of the libvirt project, which aims to provide a toolkit to manage virtualisation platforms ^1. We can use the following commands to find and manage KVM guests:

  • To list all the available guests, we can use: virsh --connect=qemu:///system list --all
  • To start a given guest, we can use: virsh --connect=qemu:///system start NAME_OF_GUEST
  • To power off a given guest, we can use: virsh --connect=qemu:///system shutdown NAME_OF_GUEST

You may wonder why we need --connect=qemu:///system part in the virsh command above. This is because normally, we would need to run virsh as root (so with sudo). However, if you are a member of the libvirt group, you can use the --connect=qemu:///system. This make things a bit more convenient, especially when we need to use the command in a script.
